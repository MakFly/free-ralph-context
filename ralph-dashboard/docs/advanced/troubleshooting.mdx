# Troubleshooting

Solve common Ralph Dashboard issues.

## Connection Issues

### Dashboard not loading

**Symptoms**: Blank page, connection errors

**Solutions:**

1. Check if server is running:

   ```bash
   lsof -i :3847
   ```

2. Restart the server:

   ```bash
   ralph dashboard restart
   ```

3. Check firewall settings

### SSE not connecting

**Symptoms**: No real-time updates

**Solutions:**

1. Verify SSE endpoint:

   ```bash
   curl http://localhost:3847/events
   ```

2. Check browser console for errors

3. Ensure no proxy is blocking SSE

## Data Issues

### Projects not showing

**Symptoms**: Empty project list

**Solutions:**

1. Verify Claude Code paths:

   ```bash
   echo $RALPH_SESSIONS_PATH
   echo $RALPH_TRANSCRIPTS_PATH
   ```

2. Check file permissions:

   ```bash
   ls -la ~/Library/Application\ Support/Claude/
   ```

3. Trigger manual refresh:
   ```bash
   curl http://localhost:3847/refresh
   ```

### Incorrect token counts

**Symptoms**: Token usage seems wrong

**Solutions:**

1. Check data source indicator
2. Verify transcript files are readable
3. Re-index transcripts:
   ```bash
   ralph reindex
   ```

## Performance Issues

### Dashboard lagging

**Symptoms**: Slow UI, high CPU

**Solutions:**

1. Increase refresh interval:

   ```json
   { "refreshInterval": 1000 }
   ```

2. Reduce project count (archive old ones)

3. Check browser extensions

### High memory usage

**Symptoms**: Browser using >1GB RAM

**Solutions:**

1. Clear old checkpoints
2. Reduce metrics retention
3. Restart browser periodically

## Compression Issues

### Compression failed

**Symptoms**: Error when compressing

**Solutions:**

1. Check AI service status
2. Verify context is not already compressed
3. Try with lower target ratio:
   ```typescript
   ralph_compress({ targetRatio: 0.5 })
   ```

### Too much removed

**Symptoms**: Important content lost

**Solutions:**

1. Restore from checkpoint
2. Adjust preservation settings
3. Report the issue

## Debug Mode

Enable verbose logging:

```bash
RALPH_DEBUG=true
RALPH_LOG_LEVEL=debug
ralph dashboard start
```

Check logs:

```bash
tail -f ~/.ralph/ralph.log
```

## Getting Help

If issues persist:

1. Check [GitHub Issues](https://github.com/yourusername/ralph-dashboard/issues)
2. Join Discord community
3. Create a bug report with:
   - Ralph version
   - OS and version
   - Steps to reproduce
   - Error logs

> **Warning:**
> **Before reporting**: Always try the latest version first.
