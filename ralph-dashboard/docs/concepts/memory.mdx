# Memory System

The Memory system stores important information across sessions.

## What are Memories?

Memories are structured pieces of information that Ralph preserves:

- **Decisions**: Architecture choices, design decisions
- **Actions**: Bug fixes, refactorings completed
- **Errors**: Problems encountered and solutions
- **Progress**: Milestones and achievements
- **Context**: Project-specific knowledge

## Memory Categories

```typescript
type MemoryCategory =
  | 'decision' // Architectural decisions
  | 'action' // Actions taken
  | 'error' // Errors fixed
  | 'progress' // Progress updates
  | 'context' // Context information
  | 'other' // Everything else
```

## Creating Memories

### Manual Creation

```typescript
ralph_add_memory({
  content:
    'Chose TanStack Table over react-table for better TypeScript support',
  category: 'decision',
  priority: 'high',
})
```

### Automatic Creation

Ralph automatically creates memories for:

- High-priority errors
- Significant decisions
- Completed features
- Performance optimizations

## Memory Attributes

```typescript
interface Memory {
  id: string
  content: string
  category: MemoryCategory
  priority: 'high' | 'normal' | 'low'
  project: string
  createdAt: string
  accessCount: number
}
```

## Searching Memories

```typescript
// Semantic search
const results = ralph_search_memories('database schema')

// Filter by category
const decisions = ralph_get_memories({ category: 'decision' })

// Get high-priority memories
const important = ralph_get_memories({ priority: 'high' })
```

> **Tip:**
> **Star important memories** for quick access. They appear at the top of search results.

## Memory Lifecycle

1. **Creation**: Added during conversation
2. **Access**: Retrieved when relevant
3. **Update**: Can be modified over time
4. **Archival**: Old memories are compressed
